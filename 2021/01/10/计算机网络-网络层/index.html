概念

网络层的主要任务：把分组从源端口传到目的端口，为分组交换网上的不同主机提供服务。
网络层的传输单位是数据报

网络层功能

  • 功能一：路由选择与分组转发
  • 功能二：异构网络互联
  • 功能三：拥塞控制==>分为开环控制（静态）和闭环控制（动态）

数据交换方式（重点）

网络层的核心是路由器

电路交换

典型：打电话

简要：一个用户独占一条电路资源，采用多路复用技术（几条电路捆在一起）

报文交换


交换设备会根据情况选择下一个要转发的交换设备，比起上一种方法线路利用率高而且灵活

分组交换

跟报文一样采用存储转发的技术，但是会进行分组（将大的数据块分割成小的数据块）

缺点二中传输额外的信息量指的是要将报文排序

数据报方式&虚电路方式

数据报方式为网络层提供无连接服务
虚电路方式为网络层提供连接服务

无连接服务：不事先为分组的传输确定传输路径，每个分组独立确定传输路径，不同分组的传输路径可能不同
连接服务：首先为分组的传输确定路径（建立连接），然后沿该路径（连接），系列分组传输路径相同，传输结束拆除连接


数据报和虚电路的对比

报文交换&分组交换


一般分组交换的时延比报文交换小，因此现实采用分组交换

三种数据交换方式的总结


电路交换：先建立电路连接、然后独占线路
报文交换：无需建立连接，整个报文发送，采用存储转发
分组交换：先将报文分组，无需建立连接，采用存储转发，时延一般小于报文交换

路由协议

由于需要区分内外网络，且某些单位不想让外界知道自己的路由选择协议，因此需要自治系统
自治系统AS：在单一的技术管理下的一组路由器，而这些路由器使用一种AS内部的路由选择协议和共同的度量以确定分组在该AS内的路由，同时还使用一种AS之间的路由协议以确定在AS之间的路由。一个AS内的所有网络都属于一个行政单位来管辖，一个自治系统的所有路由器在本自治系统内都必须连通。

  • 路由选择协议分为内部网关协议（AS内使用RIP、OSPF）、外部网关协议（AS内使用BGP）

RIP协议

概念

  • 基于距离的路由选择算法，简单
  • RIP协议要求每个路由器都维护从自身到每个目的网路的唯一最佳距离记录
  • 距离：一般指“跳数”，表示从源端口到目的端口经过的路由器个数，特殊的，直接相连的路由器距离是1。
  • RIP允许一条路由最多包含15个路由器，因此距离>=16则表示网络不可达
  • RIP的特性限制它只适用小型互联网

实现

RIP协议实现：

  1. 仅和相邻路由器交换信息
  2. 路由器交换的信息时自己的路由表
  3. 每30秒交换更新次路由信息，超过180s没有收到相邻路由器的通告，默认不可达，更新路由表
  4. 所有路由器都知道到达本自治系统任何一个网络的最短距离和下一跳路由器的地址，即“收敛”

OSPF协议

概念：OSPF指开放最短路径优先OSPF协议，来源于Dijkstra提出的最短路径算法SPF
特征：分布式的链路状态协议

实现

  • 使用洪范式向自治系统内所有路由器，即路由器向每一个相邻的路由器发送信息，然后每一个相邻的路由器又将该信息发送出去，最终区域内每个路由器都有该消息的副本
  • 发送的信息就是与本路由器相邻的所有路由器的链路状态
  • 只有当链路发生改变的时候，路由器才会开始发送信息

最后，所有路由器都有一个链路状态数据库，即全网拓扑图

链路状态路由算法

  1. 每个路由器使用【HELLO问候分组】，并了解邻居节点的网络地址
  2. 设置到它的每个邻居的成本度量metric
  3. 够到【DD数据库描述分组】，向邻站给出自己的链路状态数据库中的所有链路状态项目的摘要信息
  4. 如果DD分组中的摘要自己都有，则邻站不做处理；如果没有或者摘要更新，就发送【LSR链路状态请求分组】，请求新的信息
  5. 收到LSR后，发送【LSU链路状态更新分组】更新
  6. 更新后，发送【LSAck链路状态确认分组】进行确认
  7. 使用Dijkstra算法将根据链路状态数据库构造到其他节点的最短路径

BGP-4的四种报文

  1. OPEN（打开）报文：与相邻的BGP发言人建立关系，并认证发送方
  2. UPDATE（更新）报文：通告新路径或撤销原路径
  3. KEEPALIVE（保活）报文：没有UPDATE报文的时候，通过这种报文周期性地确认邻站的连通性；也作为OPEN的确认
  4. NOTIFICATION（通知）报文：报告前面报文的错误；也可以用来关闭连接

名词解释：

  1. 版本：使用IPv4还是IPv6
  2. 首部长度：单位是4B,最小是5
  3. 区分服务：指示期望获取哪种类型的服务
  4. 总长度：首部+数据的长度，单位是1B
  5. 标识：同一数据报的分片使用同一标识
  6. 标志：只有两位有意义，x__。中间位DF（Don’t Fragment），DF=1，禁止分片;DF=0,允许分片。最低位MF（More Fragment）：MF=1，后面还有“分片”；MF=0，代表此片为最后一片

  8. 生存时间（TTL）：IP分组的保质期，每经过一个路由器-1
  9. 协议：数据部分的协议
  10. 首部校验和：只校验首部
  11. 源IP地址和目的地址：32位
  12. 可选字段：0~40B，用来拍错、测量及安全等措施
  13. 填充：全0，把首部补成4B的整数倍

ICMP协议


ICMP全称网络控制报文协议，支持主机或路由器：差错报告、网络探询

ICMP差错报告报文

  1. 终点不可达（不能交付数据时就向源点发送终点不可达报文）
  2. 源点抑制（路由器或主机拥塞时，就向源点发送源点抑制报文，使源点知道应当把数据报的发送速率放慢）
  3. 时间超过（路由器收到TTL=0的数据报时，向源点发报；终点超过规定时间没有接收到一个数据报的全部片时，向源点发报）
  4. 参数问题（路由器或主机收到的数据报的首部字段有的字段有问题时，丢弃数据报，并向源点发送参数问题报文）
  5. 改变路由（重定向）（路由器发送此报通知主机选择更好的路由）

  • 格式

  • 不发送ICMP差错报文的情况

  1. 对ICMP差错报告报文不发送ICMP差错报告报文
  2. 对第一个分片的数据报片的后续数据报片不发送（只发一次）
  3. 对组播地址的数据报不发送
  4. 对特殊地址（127.0.0.1或0.0.0.0）不发送

ICMP询问报文

  1. 回送请求和回答报文（主机或路由器向特定主机发出询问，收到报文的主机需要发送ICMP回送回答报文给源点）测试目的站点是否可达或了解相关状态
  2. 时间戳请求和回答报文（请求主机或路由当前的时间）同步时钟和测量时间
  3. 掩码地址请求和回答报文（不使用）
  4. 路由器询问和通告报文（不使用）

应用

  1. PING命令，使用了ICMP回送请求和回答报文
  2. Traceroute命令，使用了ICMP时间超过差错报告报文，逐步减小TTL，可以得到路径上的路由器信息

QoS：Quality of Service，服务质量，指一个网络能够利用各种基础技术，为指定的网络通信提供更好的服务能力，是网络的一种安全机制，是用来解决网络延迟和阻塞问题的一种技术

名词解释：

  1. 版本：指明协议版本，此处总是为6
  2. 优先级：区分数据报的类别和优先级

IPv6和IPv4的区别

IPv6：

IPv4:

  1. IPv6将地址从32位（4B）扩大到了128位（16B），更大的地址空间
  2. IPv6将IPv4的校验和字段彻底移除，以减少每跳的处理时间
  3. IPv6将IPv4的可选字段移出首部，变成了拓展首部，路由器通常不对拓展首部进行检查，提高路由器处理效率
  4. IPv6支持即插即用，不需要DHCP协议
  5. IPv6只能在主机处分片，IPv4可以在主机和路由器处分片
  6. IPv6必须是8B的整数倍，IPv4的首部是4B的整数倍
  7. ICMPv6：附加报文类型“分组过大”，由于只能在主机分片，因此可能不符合路由器的需求
  8. IPv6支持资源的预分配，支持实时视像等要求，保证一定的带宽和时延的应用
  9. IPv6取消了协议字段，改成了下一个首部字段
  10. IPv6取消了总长度字段，改用有效载荷字段
  11. IPv6取消了服务类型字段

IPv6地址的表示形式

  • 一般形式
    冒号十六进制：4BF5：AA12：0216：FEBC：BA5F：039A：BE9A：2170
  • 压缩形式
  1. 多个0可以变成一个0
    4BF5：0000:0000:0000：BA5F:039A:000A:2176
    变成
    4BF5:0:0:0:BA5F:39A:A:2176
  2. 零压缩：一连串的零被一堆冒号取代
    FF05：0:0:0:0:0:0：B3
    变成FF05::B3

零压缩只能使用一次，意味着压缩表示只能有一对冒号

IPv6的基本地址类型

  1. 单播：一对一通信，可作为源地址+目的地址
  2. 多播：一对多通信，可作为目的地址
  3. 任播：一对多中的一个通信，可作为目的地址

IPv6向IPv4过渡的策略

IP数据报传输

  1. 单播：用于发送数据包到单个目的地，是点对点传输方式
  2. 广播：发送数据包到同一个广播域或子网内的所有设备的数据传输方式，是点对多点传输方式
  3. 组播（多播）：发送给需要特定数据的某些用户，是点对多点的传输方式

单播占用网络资源多，因此需要一条数据通路
组播提高了数据传送效率，需要组播路由器（运行组播协议的路由器）

IP组播地址

IP组播地址让源设备能将分组发送给一组设备。属于多播组的设备将被分配一个组播组IP地址（一群共同需求的主机的相同标识）

  1. 组播地址范围为224.0.0.0~239.255.255.255（D类地址），一个D类地址表示一个组播组。只能用作分组的目的地址，不能用作源地址，源地址总为单播地址
  2. 组播采用UDP协议
  3. 组播不产生ICMP报文
  4. 并非所有D类地址都可以作为组播地址

可能会考察将IP组播地址映射为MAC地址

IGMP协议


IGMP和ICMP一样都使用IP数据报传输报文

工作阶段

  • 第一阶段

    某个主机要加入组播组，该主机向组播组的组播地址发送一个IGMP报文，声明要加入该组
    本地组播路由器收到IGMP报文后，利用组播路由选择协议把这组成员关系发给其他组播路由器

  • 第二阶段

    本地组播路由器周期性探询本地局域网上的主机，以便知道这些主机是否还是组播组的成员
    只要有一个主机对某个组响应，那么组播路由器就认为这个组是活跃的；如果经过几次探询之后没有主机响应，组播路由器就认为本网络没有此组播组的主机，不再向其他组播路由器发送这组的成员关系

组播路由选择协议

组播路由协议目的是找出以源主机为根节点的组播转发树，避免路由器间兜圈子，同一个多播组，对不同的源点，有不同的多播转发树

常用三种算法：

  1. 基于链路状态的路由选择
  2. 基于距离-向量的路由选择
  3. 协议无关的组播（稀疏/密集）

移动IP

术语

移动IP是移动节点以固定的网络IP地址，实现跨越不同网段的漫游功能，并保证了基于网络IP的网络权限在漫游过程中不发生任何改变

  1. 移动节点：拥有永久IP地址的移动设备
  2. 归属代理（本地代理）：一个移动节点拥有的就“居所”称为归属网络，在归属网络中代表移动节点执行移动管理能力的实体叫做归属代理
  3. 外部代理（外地代理：在外部网络中帮助移动节点完成移动管理功能的实体称为外部代理
  4. 永久地址（归属地址/主地址）：移动站点在归属网络中的原始地址
  5. 转交地址（辅地址）：移动站点在外部网络使用的临时地址
