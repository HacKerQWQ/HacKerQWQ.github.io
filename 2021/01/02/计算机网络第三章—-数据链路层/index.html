基本概念

名词解释：

  • 结点：主机、路由器
  • 链路：网络中两个结点之间的物理通道，链路的传输介质主要有双绞线、光纤和微波。分为有线链路、无线链路
  • 数据链路：网络中两个结点之间的逻辑通道，把实现控制数据传输协议的硬件和软件加到链路上就构成数据链路
  • 帧：链路层的协议数据但由于，封装网络层数据报

功能概述

  • 功能一：为网络层提供服务。无确认无连接服务，有确认无连接服务，有确认面向连接服务
  • 功能二：链路管理，即连接的建立、维持、释放（用于面向连接的服务）
  • 功能三：组帧
  • 功能四：流量控制。限制发送方
  • 功能五：差错控制

透明传输是指不管传送的数据是什么比特组合，都能在链路上传送

封装方法之字符计数法


这种方法不常用，因为当首个字符出错时，后面就会全部错

封装方法之字符填充法


这种方法用SOH作为帧的开始，用EOT作为帧的结尾，当中间出现了假EOT时在前面加一个ESC起到类似转义的作用，对方接收帧之后再去除就可以了

封装方法之零比特填充法


这种方法跟字符填充法一样采用标志头和标志尾，不同的是标志头和标志尾是一样的01111110,当中间出现5个1时，就在后面加入1个0，接收方再反向去除就可以了
保证了透明传输：在传输的比特流中可以传送任意的比特组合，而不会引起对帧边界的判断错误

差错控制

差错概念

差错都是由噪声引起

  • 全局性：由于线路本身的电气特性产生的随机噪声（热噪声），是信道固有的，随机存在的。解决办法：提高信噪比来减少或避免干扰
  • 局部性：外部短暂原因产生的冲击噪声，是噪声的主要原因。解决办法：利用编码技术解决

  • *与物理层编码的不同**：物理层编码针对的是单个比特，数据链路层的编码针对的是一组比特

检错编码之奇偶校验码


D不能检测出来出错，因为本身是4个1，采用奇校验的话就要使最终的1个数为奇数，所以校验元是1，一共5个1，最后接收到的也是5个1，因此不能检错

流量控制和可靠传输机制（重点）

数据链路层的流量控制

注意：当发送速度相对接收速度高时，会造成传输出错，因此需要流量控制

  1. 数据链路层的流量控制是点对点的，传输层的流量控制是端到端的（点对点是两个具体设备之间，端到端是逻辑上的接收方和发送方）
  2. 数据链路层流量控制手段是：接收方收不下就不确认
  3. 传输层流量控制手段是：接收端给发送端一个窗口公告
  • 流量控制的方法：

    停止-等待协议低效，滑动窗口协议较高效，当收到确认帧就向前移一格

    发送窗口的大小是固定的

停止-等待协议

丢包：物理线路故障、设备故障、病毒攻击、路由信息错误等原因会导致数据报的丢失
概念：发送完一个分组就停止发送，等待对方确认，在收到确认后再发送下一分组
应用：无差错情况&有差错情况

无差错情况


这是理想状态，一般是有差错情况

有差错情况

  1. 数据帧丢失或检测到帧出错
    超过超时计时器的时间就会重传没有收到确认的帧
  2. ACK丢失

    同上，重传帧
  3. ACK迟到

    重传帧后，接收方再次发送ACK，接收方收到ACK后接着发送下一个帧，忽略后来迟到的ACK

解决办法

由于停止等待协议发送一个帧，因此为了提高传输效率就一次多发几个帧，这就用到了GBN协议和SR协议,需要缓存更多的帧和更多的帧序号

GBN协议（后退N帧协议）


1.发送方可以一次发送很多个帧，接收方一次接收一个帧
2.当接收方接收到前面的确认帧之后才向前滑动窗口（不是逐个确认再前进，而是返回了3号帧的ACK之后滑动窗口继续发送）

GBN发送方必须响应的三件事

  1. 上层的调用
    上层要发送数据时，检查窗口是否已满，未满则产生帧并排入到帧序列准备发送；如果窗口已满，则把数据返回给上层表示已满（实际中是缓存这些数据，窗口不满时再发送帧）
  2. 收到了一个ACK
    GBN协议中，对n号帧采用累积确认，表示接收方已经收到了n号帧和它之前的所有帧
  3. 超时事件
    之所以叫后退N帧/回退N帧，来源于出现丢失和时延过长时发送方的行为。当等待ACK的时间超出定时器的时间时，发送方会重传所有已发送但未被确认的帧

GBN接收方要做的事

  1. 确认帧的顺序正确且正确接收，然后将数据部分交给上层
  2. 当接收到的帧跳过某号帧时，为最近按序接收的帧重新发送ACK，接收方会将缺失帧的信息丢弃，因为它只期待：expectedseqnum（下一个按序接收的帧序号）

滑动窗口限制

小总结

  1. 累积确认（偶尔确认，不是每帧确认）
  2. 接收方只按顺序接收帧，不按序则丢弃
  3. 确认序列号最大的、按序到达的帧
  4. 发送窗口最大为2**n-1，接收窗口大小为1

  • 容易前功尽弃，只要没有收到某个ACK，则正确传送的数据帧要重传，传送效率低

SR协议（选择重传协议）

使用SR协议的接收方将已收到的帧缓存下来，等待缺失的帧，发送方发送窗口内的帧，等待每个帧的ACK，如果等到了最左边帧的ACK就向右滑动窗口

SR发送方要做的三件事

  1. 上层的调用
    接收到上层的数据之后，检查窗口内是否还有空位，有就准备发送，没有就缓存等待
  2. 收到一个ACK
    收到ACK后，将窗口内的该帧标记为已接收，如果该帧位于窗口下界就滑动窗口到未确认的最小帧处，发送新进入窗口内的有序号的帧
  3. 超时事件
    当超时事件发生后，只重传一个帧

SR接收方要做的事

  • 来者不拒（窗口内的帧）
    对于接收到的帧无论是否顺序都会接收，缓存下来并且给发送方发送ACK帧，直到所有帧（意味比失序的更小的帧）都接收到为止，接收到后，就会将这一批帧交付上层，然后滑动窗口
  • 如果收到了小于窗口序号的帧，返回一个ACK即可

CSMA协议

CSMA全称载波监听多路访问协议（carrier sense multiple access）
CS：载波侦听/监听，每一个站在发送数据之前要检测一下总线上是否有其他计算机在发送数据
MA：多点接入，表示多个计算机以多点接入的方式连接在一根总线上

协议思想：发送帧之前，监听信道

注意：坚持是指对于监听信道忙之后的坚持监听

  • 1-坚持CSMA
    思想：如果有主机要发送信息，先监听信道

    优点：只要媒体空闲，站点马上发送，避免了媒体利用率的损失
    缺点：冲突不可避免

  • 非坚持CSMA

  • *非坚持指的是监听信道忙之后不继续监听**

    优点：采用随机的重发延迟时间可以减少冲突发生的可能性
    缺点：大家都在等待的时候，媒体处于空闲状态，媒体使用率低

  • p-坚持CSMA

  • *p-坚持指的是对于监听信道空闲的处理**
    思想：主机要发送信息，先监听信道，空闲则以p概率直接传，1-p概率到下一个时间槽再传输。忙则等一个随机的时机时间再监听

    优点：集合了1-坚持CSMA的减少媒体空闲时间和非坚持CSMA的减少冲突
    缺点：发生了冲突依然坚持把数据帧发完，造成浪费

CSMA/CA协议

全称载波监听多点接入/碰撞避免CSMA/CA（carrier sense multiple access with collision avoidance）

  • CSMA/CA存在的原因：
  • CSMA/CA工作原理：
  1. 发送数据前，检测信道是否空闲
  2. 空闲就发出RTS(request to send),RTS包括发射端的地址、接收端的地址、下一份数据将持续发送的时间等信息，信道忙则等待
  3. 接收端收到RTS后，响应CTS(clear to send)
  4. 发送端收到CTS后，开始发送数据帧（同时预约信道，告知其他站点自己要传多久）
  5. 接收端收到数据震后，用CRC检验数据是否正确，正确则响应ACK帧
  6. 发送方收到ACK就准备下一个数据帧的发送，没有则一直重传直到达到规定重发次数为止（采用二进制指数规避算法确定随机的推迟时间）

MAC子层和LLC子层

IEEE 802将数据链路层分为LLC子层和MAC子层

  • LLC子层负责识别来自上层协议（网络层），对数据进行封装，LLC报头告诉数据链路层一旦帧被接收到，应当对数据包如何处理。简而言之，为上层提供服务
  • MAC子层主要功能包括数据帧的封装/卸装、帧的寻址和识别、帧的接收和发送、链路的管理、帧的差错控制等

局域网

  • 局域网简称LAN，值某一区域内由堕胎计算机互联成的计算机组，使用广播信道

特点：

  1. 覆盖范围小
  2. 有专门铺设的传输介质（双绞线，同轴电缆），传输速率高
  3. 通信延迟端，误码率低，可靠性高
  4. 各站为平等关系，共享信道
  5. 多采用分布式控制和广播式通信，能进行广播和组播

以太网

概念：由Xerox公司创建，由Xerox、Intel和DEC公司联合开发的基带总线局域网规范，是当今局域网最通用的通信协议标准，采用CSMA/CD技术

以太网技术具有统治性地位的原因：

  1. 造价低
  2. 应用广泛
  3. 比令牌环网、ATM网便宜，简单
  4. 满足网络速率要求：10Mb/s~10Gb/s

两个以太网标准

  1. DIX Ethernet V2:第一个局域网产品（以太网）规约
  2. IEEE 802.3 ：IEEE 802委员会802.3工作组制定的第一个IEEE的以太网标准

两个以太网标准基本等价

  • 特点：
  1. 无连接：发送方和接收方之间没有“握手过程”
  2. 不可靠：不对发送方的数据帧编号，接收方不向发送方进行确认，差错帧直接丢弃，差错纠正由高层负责。
  • 以太网只有无差错接收，不实现可靠传输

  1. 有固定的基础设施无线局域网
  2. 无固定基础设施无线局域网的自组织网络

广域网

广域网（WAN，Wide Area Network），跨接很大的物理范围

  • 通信子网主要使用分组交换的形式，将分布在不同地区的局域网和计算机系统互连起来，达到资源共享的目的

PPP协议

PPP(Point-to-Point Protocol)是使用最广泛的数据链路层协议，只支持全双工链路

PPP需要满足的要求：

  • 简单（对于链路层的帧，无需纠错，无需序号，无需流量控制）
  • 封装成帧（帧定界符）
  • 透明传输（与帧定界符一样比特组合的数据应该如何处理：异步线路用字节填充，同步线路用比特填充）
  • 多种网络层协议（封装的IP数据报可以采用多种协议）
  • 多种类型链路（串行/并行，同步/异步，电/光）
  • 差错检测（错就丢弃）
  • 检测连接状态（链路是否正常工作）
  • 最大传送单元（数据部分最大长度MTU）
  • 网络层地址协商（需要知道通信双方的网络层地址）
  • 数据压缩协商

PPP协议无需满足的要求：

  • 纠错
  • 流量控制
  • 序号
  • 不支持多点线路

PPP的三个组成部分

  1. 一个将IP数据报封装到串行链路（同步串行/异步串行）的方法
  2. 链路控制协议LCP：简历并维护数据链路连接。
  3. 网络控制协议NCP：PPP可支持多种网络层协议，每个不同的网络层协议都要一个相应的NCP来配置，为网络层协议简历和配置逻辑连接

LCP和NCP流

PPP协议帧格式


第三个部分目前没有得到应用，PPP协议帧面向字节

HDLC协议

HDLC（High-Level Data Link Control或简称HDLC），是一个在同步网上传输数据、面向比特的数据链路层协议，采用全双工通信

  • 所有帧采用CRC检验，对信息帧进行顺序编号，可防止漏收或重发，传输可靠性高
  • 主站主要功能是发送命令，接收响应帧，负责对整个链路的控制系统的初启、流程的控制、差错检测或恢复等。
  • 从站的主要功能是接收来自主站的命令帧、向主站发送响应帧，配合主站参与差错恢复等链路控制
  • 复合站的主要功能是发送、接收命令帧和响应帧，负责整个链路控制

三种数据操作方式：

  1. 正常响应方式（从站接收到来自主站的命令帧再执行命令）
  2. 异步响应方式（每个站是平等的地位，不需要等待其他站的命令帧）
  3. 异步响应方式（从站可以在没有接收到主站命令帧的情况下执行命令）

HDLC的帧格式


主要记住HDLC的控制帧有信息帧、监督帧、无编号帧

PPP协议和HDLC的相同点和不同点

  • HDLC和PPP只支持全双工
  • 都可以实现透明传输
  • 都可以实现差错控制，但不纠错

链路层设备

网桥

网桥是交换机的前身

透明网桥

概念：“透明”是指以太网上的站点并不知道发送的帧经过哪几个网桥

工作原理：找目标MAC地址的时候，如果网桥没有源的地址，会在转发表添加记录，然后广播式地发送到另外的端口

源路由网桥


在通过路由的时候，帧头部带有路由信息

多接口网桥–以太网交换机


也就是我们现在常用的交换机

  • 直通式交换机
  1. 查完目的地址（6B）就立即转发
  2. 延迟小，可靠性低，无法支持具有不同速率的端口的交换
  • 存储转发式交换式
  1. 将帧放进高速缓存，检查是否正确，正确就转发，错误就丢弃
  2. 延迟大，可靠性高，可以具有不同速率的端口的交换

冲突域&广播域

冲突域：同一时间只有一台设备发送信息的范围
广播域：如果一个站点发出一个广播信号，所有能接收收到这个信号的设备范围称为一个广播域

例题


4个冲突域，一个广播域
解析：交换机一个口就是一个冲突域，物理层设备连接的就是一个广播域
